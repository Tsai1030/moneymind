# MoneyMind 貓咪互動設計計畫書

> 版本:v1.0(概要版）
> 主題:以 3D 貓咪為核心的擬人化記帳體驗
> 使用模型:Cute Cat Model（圓潤胖胖款，Sketchfab，CC-BY 授權）

---

## 一、專案概述

MoneyMind 是一款記帳／理財 Web App。本計畫的目標是把「冷冰冰的數字管理」轉化為「養一隻貓的情感陪伴」,透過一隻 3D 貓咪角色,讓使用者因為「想照顧貓」而養成記帳習慣。

核心理念:**貓咪的狀態 = 使用者的財務健康**。存得好,貓開心;超支,貓擔心;不記帳,貓睡著。讓財務狀況「被看見」,比看數字更直覺。

---

## 二、貓咪角色設定

| 項目 | 設定 |
|---|---|
| 角色定位 | 理財夥伴 / 財務小秘書 |
| 暫定名稱 | 由使用者在 onboarding 時命名(預設「錢錢」) |
| 個性 | 溫柔、會撒嬌、偶爾碎念 |
| 說話口吻 | 擬人化貓秘書,句尾帶「喵」,例:「主人這個月罐罐快吃完了喵～」 |
| 視覺風格 | 圓潤、大圓角、暖色系(奶油白＋橘貓＋薄荷綠) |

---

## 三、貓咪情緒系統(核心機制)

依「本月剩餘預算 ÷ 剩餘天數」計算貓咪心情,共 4 種狀態:

| 心情 | 觸發條件 | 貓咪表現 |
|---|---|---|
| 😺 開心 | 預算充裕、記帳正常 | 毛色發亮、搖尾巴、主動討摸 |
| 😻 興奮 | 達成存錢目標 | 跳躍、撒彩帶慶祝 |
| 😿 擔心 | 接近或超出預算 | 垂耳、躲起來、發出哀求聲 |
| 😴 睡著 | 連續數日未記帳 | 趴著睡、灰濛濛、需被喚醒 |

> 表情若模型不支援 morph target,改用「頭頂浮出 emoji 泡泡 + 動畫切換」來表現。

---

## 四、頁面 3D 分配原則

核心原則:**3D 服務情緒,2D 服務效率**。

| 頁面 | 3D 程度 | 說明 |
|---|---|---|
| Landing / 登入 | 全螢幕 3D | 第一印象,貓咪登場 |
| Dashboard 主頁 | 上半 3D + 下半 2D | 上半貓咪在窩裡,下半財務數據卡片 |
| 記帳輸入頁 | 純 2D | 速度優先,送出後角落小貓彈出反應 |
| 報表分析頁 | 純 2D + 角落 2D 貓 | 圖表為主,貓咪在側講評 |
| 貓咪小窩(成就頁) | 全螢幕 3D | 養成系統主場,可拖曳視角 |
| 設定 / 帳號 | 純 2D | 不干擾功能操作 |

## 五、互動細節清單(依開發優先序)

1. **情緒系統**(必做)— 4 種心情自動切換
2. **滑鼠／觸控追蹤**(必做)— 貓咪頭部跟隨游標,最有「活著」感
3. **點擊反應**(必做)— 點貓咪會喵、翻肚子、伸懶腰(3–5 種隨機)
4. **記帳成功動畫**(必做)— 貓咪探頭吃掉錢,即時回饋
5. **Idle 微動**(進階)— 閒置數秒後洗臉、打哈欠
6. **小窩成長系統**(進階)— 場景隨理財達成度升級
7. **節日皮膚**(後期)— 聖誕帽、新年紅包等

## 六、UX 主流程

### 流程 A:新手引導(Onboarding)

```
歡迎頁(3D 貓咪登場)
   ↓
幫貓咪取名字
   ↓
設定本月預算(包裝成「貓咪的罐罐目標」)
   ↓
引導記下第一筆消費
   ↓
貓咪做出開心反應
   ↓
進入 Dashboard
```

重點:命名 + 設預算其實就是 onboarding,但包裝成「養貓」,讓使用者覺得「這是我的貓」。

### 流程 B:日常使用(每天約 30 秒)

```
打開 App
   ↓
看到貓咪今日心情(= 今日財務狀態)
   ↓
點右下浮動「＋」按鈕
   ↓
2D 表單記帳
   ↓
送出時貓咪跑出來吃掉那筆錢
   ↓
回 Dashboard
```

重點:貓咪表情就是 Dashboard 的 KPI,比看數字直覺。

### 流程 C:週／月回顧

```
推播:「錢錢有話跟你說喵～」
   ↓
進入報表頁
   ↓
貓咪在側邊指著圖表講評
   ↓
達標 → 升級貓窩 + 解鎖配件
```

## 七、貓咪小窩成長系統

升級條件綁定「理財達成度」(連續記帳天數、達成存錢目標次數等),**不是花錢**,避免鼓勵消費。

| 等級 | 場景 | 解鎖條件範例 |
|---|---|---|
| Lv.1 | 紙箱 | 初始 |
| Lv.2 | 小毯子 | 連續記帳 7 天 |
| Lv.3 | 貓抓板 | 達成首次月預算 |
| Lv.4 | 貓跳台 | 累積達標 3 個月 |
| Lv.5 | 貓別墅 | 累積達標 6 個月 |

## 八、技術架構(概要)

| 層 | 技術 |
|---|---|
| 前端框架 | Next.js 14 |
| 3D 渲染 | React Three Fiber + @react-three/drei |
| 模型格式 | .glb（Draco 壓縮） |
| 模型拆解工具 | gltfjsx |
| 2D 動畫 | Lottie（角落小貓、特殊動作） |
| 效能策略 | 3D 場景 lazy load；弱機降階為 2D 版本 |

效能注意:Dashboard 的 3D 為常駐,須極輕量(低面數);手機端偵測 GPU,弱機自動退回 2D Lottie 版本。

## 九、開發階段規劃

| 階段 | 內容 | 預估 |
|---|---|---|
| Phase 1 | Dashboard 3D 貓咪 + 情緒系統 + 滑鼠追蹤 | 第 1 週 |
| Phase 2 | 接記帳邏輯 + 記帳成功反應動畫 | 第 2 週 |
| Phase 3 | 貓咪小窩成長系統 | 第 3 週 |
| Phase 4 | 報表頁講評 + 節日活動 | 之後 |

建議:**先做 Phase 1 驗證技術鏈與使用者反應,再逐步擴充**,不要一次做完。
